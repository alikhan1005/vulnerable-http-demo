### 3.4 Небезопасная десериализация
- Эндпоинт `/deserialize` использует `pickle.loads`.
- Демонстрация (python):

```python
import pickle, requests
payload = pickle.dumps({"cmd": "whoami"})
requests.post('http://127.0.0.1:5000/deserialize', data=payload)
```

*Вставь сюда вывод/скриншот.*

## 4. Анализ рисков
- Утечка секретов → компрометация внешних API
- Stacktrace → сбор информации о стекe/версиях
- config.json в репозитории → утечка учётных данных
- pickle.loads → возможность удалённого выполнения кода (RCE)

## 6. Вывод и рекомендации
- Никогда не хранить секреты в коде/репозиториях
- Использовать безопасные форматы (JSON) и валидацию
- Логи ошибок — в log-систему, пользователю — общий ответ
- Регулярно сканировать репозиторий на утечки (git-secrets, truffleHog)
